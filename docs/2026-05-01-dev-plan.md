# JiSpec 2026-05-01 Development Plan

Date: 2026-05-01

Today is a north-star execution day. The goal is not to broaden the surface area, but to make the proven V1 mainline easier to measure, operate, and review.

Primary direction:

```text
takeover quality quantification -> execute mediation experience -> Console read-only control room
```

North-star anchor:

> raw evidence for machines, distilled decision packets for humans

## Today Goal

By the end of today, JiSpec should have a clearer way to answer:

- Is legacy takeover quality improving?
- Where does a takeover still need human correction?
- Can execute mediation tell a human or external coding tool what to do next?
- Can a future Console read the local JiSpec state without replacing CLI gates?

### T1.2 Add Retakeover Quality Metrics

Status: completed

Scope:

- Extend retakeover metrics with review-oriented quality signals.
- Keep metrics deterministic and local; no LLM scoring in the gate.

Candidate metrics:

| Metric | Purpose |
| --- | --- |
| `noiseSuppressionRate` | Shows whether dependency, build, generated, cache, and audit noise was suppressed. |
| `topEvidenceSignalRate` | Shows whether top ranked evidence is dominated by strong contract or boundary signals. |
| `adoptCorrectionLoad` | Estimates how much human correction was required during adoption. |
| `featureOverclaimRisk` | Flags cases where weak behavior evidence might be overclaimed as contract-ready. |
| `verifySafety` | Shows whether adopted/deferred takeover output stays non-blocking under verify. |
| `takeoverReadinessScore` | A conservative aggregate score for fixture-level takeover readiness. |

Expected files:

- `tools/jispec/bootstrap/retakeover-metrics.ts`
- `tools/jispec/tests/bootstrap-retakeover-regression.ts`
- `tools/jispec/tests/bootstrap-messy-legacy-takeover.ts`

Acceptance:

- Per-fixture metrics include the new scorecard fields.
- Pool metrics summarize readiness distribution across fixtures.
- Regression tests fail if noisy fixtures look falsely healthy or weak behavior gets overclaimed.

Completion note:

- `RetakeoverMetrics.qualityScorecard` now records deterministic review signals:
  `noiseSuppressionRate`,
  `topEvidenceSignalRate`,
  `adoptCorrectionLoad`,
  `featureOverclaimRisk`,
  `verifySafety`,
  `takeoverReadinessScore`,
  `riskNotes`,
  `nextAction`.
- Pool metrics now aggregate average readiness, lowest score, owner-review fixtures, blocking fixtures, and feature overclaim risk distribution.
- Focused validation passed on 2026-05-01:
  `npm run typecheck`,
  `node --import tsx ./tools/jispec/tests/bootstrap-retakeover-regression.ts`,
  `node --import tsx ./tools/jispec/tests/bootstrap-messy-legacy-takeover.ts`,
  `node --import tsx ./scripts/run-messy-legacy-takeover-demo.ts --force --json`.
